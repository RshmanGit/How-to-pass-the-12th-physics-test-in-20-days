# Coding style

> Example rule, imported by the root `CLAUDE.md`. Edit to match this project; delete if not needed.

- Defer formatting to the project's formatter/linter — don't hand-format.
- Prefer clear names over comments. Comment **why**, not **what**.
- No commented-out code in commits. Delete it; git remembers.
- Keep functions small and single-purpose.
- Match the conventions of surrounding code over personal preference.
